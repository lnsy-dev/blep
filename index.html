blep

business logic documentation generator

curl -O https://github.com/yourusername/blep/releases/latest/download/bldoc
chmod +x bldoc
./bldoc src/

quick start

1. annotate your code 
/**
 * @bl-topic Order Processing
 * @bl-detail Orders over $1000 require manager approval
 * @bl-rationale Prevents fraud and ensures oversight on large transactions
 * @bl-see Payment Processing
 */
public function createOrder($amount) {
    if ($amount > 1000) {
        // @bl-detail Manager approval required for orders exceeding $1000
        $this->requestApproval();
    }
}
2. generate documentation 
./bldoc src/ -o docs/ -t "My Project Business Logic"
3. browse output 
open docs/index.html

features

annotation system

  • @bl-topic - define top-level documentation categories
  • @bl-subtopic - create subsections within topics
  • @bl-detail - document specific business rules and workflow steps
  • @bl-rationale - explain why rules exist (appears in collapsible sections)
  • @bl-see - cross-reference related topics and create navigation links

git integration

  • automatic change history tracking via git blame
  • shows last modified date and author for each detail
  • displays recent commits affecting specific business rules
  • generates timeline changelog of all business logic changes
  • links source file locations with line numbers

search capabilities

  • full-text search powered by fuse.js
  • searches across topics, subtopics, details, and rationales
  • generates search-index.json for client-side searching
  • interactive search interface with instant results
  • no server required - works entirely in browser

output formats

  • responsive html with clean, readable layout
  • markdown versions of all documentation
  • individual topic pages with inline history
  • collapsible code snippets and rationale sections
  • cross-reference links between related topics
  • source file references with line numbers

zero dependencies

  • pure php 7.4+ with no external libraries
  • single-file distribution available
  • no php extensions required beyond standard installation
  • works on any system with php installed
  • no database or configuration files needed

command-line options

  • -o, --output - specify custom output directory
  • -t, --title - set documentation site title
  • --exclude - exclude directories (repeatable for multiple patterns)
  • -v, --verbose - show each file as it's parsed
  • --version - display version information
  • -h, --help - show usage instructions

use cases

  • document business rules where they're implemented
  • keep documentation synchronized with code changes
  • generate browsable sites for non-technical stakeholders
  • onboard new developers with business context
  • track evolution of business logic over time
  • maintain compliance and audit trails
  • understand rationale behind legacy decisions
